# Design System

## 1. Design Philosophy

### Core Principle

The portfolio should feel like a premium software product built by an experienced engineer rather than a trendy portfolio template.

The design should communicate:

- Professionalism
- Engineering craftsmanship
- Attention to detail
- Technical maturity
- Simplicity
- Timelessness

The website should not rely on excessive visual decoration. Its quality should come from:

- Typography
- Layout rhythm
- Whitespace
- Content hierarchy
- Subtle motion
- Consistent components

# 3. Design Principles

## 3.1 Avoid AI-Generated Portfolio Style

Never use:

- Large gradient blobs
- Neon colors
- Glassmorphism
- Random floating shapes
- Excessive shadows
- 3D cards
- Continuous animations
- Particle backgrounds
- Excessive borders
- A card for every piece of information

Avoid making every section visually identical.

Bad example:

```
Section Title

[ Card ]
[ Card ]
[ Card ]

----------------

Section Title

[ Card ]
[ Card ]
[ Card ]
```

This creates a template feeling.

---

## 3.2 Create Visual Rhythm

Every section should have a different presentation style.

Examples:

### Hero

Large typography + professional photograph + key metrics.

### Experience

Editorial timeline or structured professional history.

### Projects

Visual showcase with large screenshots.

### Skills

Minimal categorized skill groups.

Do not use 25 individual technology cards.

### Research

Premium publication cards.

### Education & Achievements

Compact information blocks.

### Certifications

Horizontal slider representing continuous learning.

Use a neutral foundation with a single strong accent.

Avoid using many colors.

The design identity will be based on:

- Prussian Blue
- Orange
- Neutral blacks and whites

Use subtle shadows only.

Dark mode:

Light mode:

# 10. Component Guidelines

## Navigation

Style:

Minimal sticky navigation.

Behavior:

- Transparent initially
- Background blur on scroll
- Thin border appears while scrolling

Include:

- Logo / KSK identity
- Navigation links
- Theme toggle
- Resume button

---

# Buttons

## Primary Button

Style:

Orange background.

Text:

Dark text.

Purpose:

Main call-to-actions.

Examples:

- Download Resume
- Contact

---

## Secondary Button

Style:

Transparent with border.

Purpose:

Less important actions.

Examples:

- View Projects
- View GitHub

---

# Cards

Cards should not be used everywhere.

When cards are required:

- Thin border
- Subtle surface difference
- 16px radius
- Proper spacing

Avoid creating a dashboard appearance.

---

# 11. Images & Graphics

## Profile Image

Use a real professional photograph.

Avoid:

- AI-generated portraits
- Stock photos

Image treatment:

- Clean crop
- Subtle background treatment
- Natural shadows

---

## Project Images

Always use:

- Real screenshots
- Actual application UI

Avoid:

- Placeholder letters
- Fake dashboards
- Generic illustrations

---

# 12. Icons

Use a consistent icon library.

Recommended:

- Lucide React

Avoid mixing:

- Different icon styles
- Colored icons

Icons should inherit the theme colors.

---

# 13. Motion & Interactions

## Animation Philosophy

Motion should support usability, not become decoration.

---

## Allowed Animations

- Fade in on scroll
- Subtle slide up
- Image zoom on hover
- Button hover transitions
- Navigation transitions
- Smooth theme switching

Animation duration:

200ms - 300ms

---

## Optional Premium Interactions

Use carefully:

- Magnetic buttons
- Cursor spotlight (dark mode only)
- Text reveal animations
- Smooth page transitions

---

## Avoid

- Parallax backgrounds
- Floating objects
- Continuous animations
- Excessive motion

---

# 14. Responsive Design

Mobile is not a smaller desktop.

Each section should be designed specifically for mobile.

Priorities:

- Readability
- Comfortable spacing
- Touch-friendly buttons
- Reduced visual complexity

---

# 15. Accessibility

The design must support:

- Proper contrast ratio
- Keyboard navigation
- Visible focus states
- Semantic HTML
- Screen reader compatibility

---

# 16. Signature Identity

The portfolio must contain subtle personal branding.

Possible identity elements:

- KSK logo
- Consistent orange accent line
- Custom section numbering
- Unique micro interactions

The user should remember the portfolio after leaving it.

---

# 17. Final Design Goal

When someone opens the website, they should feel:

"This is not a portfolio template. This is a carefully designed digital product built by a software developer who understands both engineering and user experience."

The final design should remain relevant and professional even after 5 years.
